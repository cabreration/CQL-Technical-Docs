CQL

Introducción DDL DML DCL TCL

Introducción

CQL significa "Compi Query Language" y es un clon del popular lenguaje de bases de datos SQL (pronunciado "Si-Cu-El") que tiene el objetivo de ayudar al aprendizaje de manipulación de bases de datos a personas que sean ajenas a las tecnologías de información.

CQL es un lenguaje de consultas que permite la manipulación de la información almacenada en un servidor de bases de datos. Este lenguaje, aunque trabaja como uno solo, se divide en 5 grandes grupos, los cuales son:

  • Lenguaje de Definición de Datos (DDL): Permite llevar a cabo las tareas de definición de las estructuras que almacenarán los datos.

  • Lenguaje de Manipulación de datos (DML): Permite la consulta, agregación, modificación o eliminación de los datos dentro de las estructuras.

  • Lenguaje de Control de Datos (DCL): Se encarga de la administración de usuarios y permisos.

  • Lenguaje de Control de Transacciones (TCL): Garantiza la integridad de la información por medio de las sentencias commit y rollback.

  • Lenguaje de Control de Flujo (FCL): Agrupa todas las sentencias de transferencia, selección y repetición, así como el manejo de variables, estructuras, entre otros.

DDL

DDL es el lenguaje de programación propio de CQL que permite llevar a cabo las instrucciones para la definición de las estructuras que almacenarán los datos, como la creación de bases de datos y tablas.

DDL es conformado por varias instrucciones:

  • Crear Bases de Datos:
    Create Database [If Not Exists] -Nombre_BD-

  • Usar una Bases de Datos:
    Use -Nombre_BD-

  • Eliminar una Base de Datos:
    Drop Database -Nombre_BD-

  • Crear una Tabla:
    Create Table [If Not Exists] -Nombre_Tabla- ( Definition_Tabla );

  • Eliminar una Tabla:
    Drop Table [If Exists] -Nombre_Tabla-;

  • Truncar una Tabla:
    Truncate Table -Nombre_Tabla-;

DML

DML significa en inglés "Data Manipulation Language", lo que se traduce como "Lenguaje de Manipulación de Datos". Este sublenguaje incluye las sentencias que permiten manipular la información almacenada en la base de datos.

Las sentencias que soporta CQL son las siguientes:

  • Insertar en una Tabla:
    Insert into -Nombre_Tabla- Values ( -Lista_Valores- );

  • Actualizar un registro:
    Update -Nombre_Tabla- Set -Lista_Valor_Atributo- [Where Expresion];

  • Eliminar un registro:
    Delete From -Nombre_Tabla- [Where Expresion];

  • Seleccionar:
    Select -Lista_Campos- From -Nombre_Tabla-;

DCL

DCL significa en inglés "Data Control Language", que se traduce como "Lenguaje de Control de Datos". Este sublenguaje se encarga de controlar el acceso a la información contenida en la base de datos.

Las instrucciones soportadas son:

  • Crear un usuario:
    Create User -Nombre_Usuario- With Password -Clave-

  • Conceder permisos:
    Grant -Nombre_Usuario- On -Nombre_BD-;

  • Revocar permisos:
    Revoke -Nombre_Usuario- on -Nombre_BD-;

TCL

Significa en inglés "Transaction Control Language" y se traduce como "Lenguaje de Control de Transacciones".

Es el lenguaje utilizado para manejar las transacciones en la base de datos. Son utilizados para manejar los cambios que se realizan a la información de la base de datos. Consta de 2 sentencias básicas:

  • Comprometer los cambios:
    Commit;

  • Deshacer los cambios:
    Rollback;
